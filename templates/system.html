系统架构

整体架构概述

本项目是一个基于 FastAPI 构建的 AI 智能助手系统，采用典型的三层架构设计：表示层、业务逻辑层和数据访问层。系统集成了大语言模型（LLM）和检索增强生成（RAG）技术，为用户提供智能对话服务。

用户认证模块

功能描述：
  • 用户注册：邮箱、密码、用户名创建账户
  • 用户登录：邮箱密码验证，返回JWT令牌
  • 令牌刷新：自动续期机制
  • 密码管理：支持密码重置（预留接口）
用户流程：
访问登录页 → 输入凭证 → 验证成功 → 获取令牌 → 跳转首页
async def authenticate_user(db: Session, email: str, password: str): user = db.query(models.User).filter(models.User.email == email).first() if not user or not verify_password(password, user.hashed_password): return False return user

用户管理模块

功能描述：
  • 个人信息查看/编辑
  • 头像上传（支持JPG/PNG/WEBP，10MB限制）
  • 多尺寸头像生成（original/large/medium/small）
  • 旧头像文件自动清理
支持的头像尺寸：
sizes = { "original": None, "large": (200, 200), "medium": (100, 100), "small": (50, 50) }

聊天对话模块

功能描述：
  • 创建/删除对话会话
  • 发送消息并获取AI响应
  • 查看对话历史
  • 会话列表管理
  • 语音输入（浏览器 Web Speech API，支持断点续录、中断检测与自动恢复）
语音输入技术细节：
voiceState: idle → listening → paused → resumed silenceThreshold: 1500ms（中断检测阈值） accumulatedTranscript: 跨 session 持久化缓存 视觉反馈: 录音(红色脉冲) → 暂停(琥珀闪烁) → 恢复(绿色闪光) 听觉反馈: Web Audio API 生成中断/恢复提示音
对话数据模型：
# Conversation: 会话实体 # ChatMessage: 消息实体（关联会话和用户） # 消息类型：user(用户消息), assistant(AI响应), system(系统消息)

RAG检索流程

# 1. 用户提问向量化 query_embedding = embedding.encode(query) # 2. 向量数据库检索 results = vector_store.similarity_search(query_embedding, k=3) # 3. 构建prompt上下文 context = "\n".join([doc.page_content for doc in results]) # 4. 调用LLM生成响应 response = deepseek_client.generate(context + query)

数据库实体关系图

erDiagram
    USER ||--o{ CONVERSATION : creates
    USER ||--o{ KNOWLEDGE_BASE : owns
    USER ||--o{ OPERATION_LOG : performs
    CONVERSATION ||--o{ CHAT_MESSAGE : contains
    KNOWLEDGE_BASE ||--o{ VECTOR_STORE : stores_in
    
    USER {
        int id PK "用户ID"
        string email UK "邮箱"
        string hashed_password "加密密码"
        string username "用户名"
        string avatar_url "头像URL"
        string role "角色: user/admin"
        boolean is_active "是否活跃"
        datetime created_at "创建时间"
        datetime updated_at "更新时间"
    }
    
    CONVERSATION {
        int id PK "会话ID"
        int user_id FK "用户ID"
        string title "会话标题"
        datetime created_at "创建时间"
        datetime updated_at "更新时间"
    }
    
    CHAT_MESSAGE {
        int id PK "消息ID"
        int conversation_id FK "会话ID"
        string role "角色: user/assistant/system"
        string content "消息内容"
        datetime created_at "创建时间"
    }
    
    KNOWLEDGE_BASE {
        int id PK "知识库ID"
        int user_id FK "用户ID"
        string filename "文件名"
        string file_path "文件路径"
        int chunk_count "分块数量"
        boolean is_shared "是否共享"
        datetime created_at "创建时间"
    }
    
    VECTOR_STORE {
        int id PK "向量ID"
        int user_id FK "用户ID"
        string content "文本内容"
        blob embedding "向量嵌入"
        json metadata "元数据"
    }
    
    OPERATION_LOG {
        int id PK "日志ID"
        int user_id FK "用户ID"
        string action "操作类型"
        string target "操作目标"
        datetime created_at "操作时间"
    }

关键模块职责

main.py - 应用入口
  • FastAPI应用实例创建
  • 路由注册
  • 中间件配置（CORS、静态文件）
  • 数据库连接初始化
database.py - 数据库配置
  • SQLAlchemy引擎创建
  • 数据库会话管理
  • 依赖注入函数
models.py - 数据模型
  • User: 用户实体
  • Conversation: 会话实体
  • ChatMessage: 消息实体
  • KnowledgeBase: 知识库实体
  • OperationLog: 操作日志

  • JWT令牌生成/验证
  • 密码加密/验证
  • 认证依赖函数
schemas.py - 数据模式
  • Pydantic模型定义
  • 请求/响应结构
  • 数据验证规则
services/ - 业务服务
  • rag_service.py: RAG核心逻辑
  • deepseek_client.py: LLM API调用
  • vector_store.py: 向量数据库操作
  • embedding.py: 文本嵌入处理

安全考虑

认证安全

认证安全措施

JWT令牌存储：HttpOnly Cookie | 密码加密：bcrypt算法（10轮）| 令牌过期机制：30分钟自动失效 | CSRF防护：预留接口

文件上传安全

文件上传安全措施

文件类型白名单验证 | 文件大小限制（知识库300MB/头像10MB）| 文件内容校验 | 存储路径随机化（UUID命名）| 禁止执行权限设置

输入验证

输入验证措施

Pydantic模型强制验证 | SQL注入防护（ORM参数化查询）| XSS防护（前端模板自动转义）| 路径遍历防护

访问控制

访问控制措施

基于角色的访问控制（RBAC）| 用户只能访问自己的资源 | 管理员权限隔离
